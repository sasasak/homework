### 1번 문제 

함수 과제를 진행하려고 1번 문제를 읽었어요 **분명 별 하나인데 왜 머릿속에 떠오르는게 없을까요** 

조건문을 아직 배우지 않았으니까 조건문을 쓰지 않고도 이 함수를 만들 수 있을텐데 

조건문을 안쓰고 도대체 어떻게 만들지???? 전혀 생각이 안나는거에요 

수업 끝나고 과제를 한 번 해볼까 하고 열어봤는데 로직이 정말 하나도 생각이 안나서 고민을 좀 해보고 만들어볼까 하면서

자기전까지 고민하다가 토요일 오후에 다시 열어봤어요 근데도 제 머리로는 알 수 없었어요 

왜냐면 모르겠으니까! !! ! 아무리 머리를 굴려봐도 제 머릿속에 있는 지식이 아니기때문에 찾을 수 없었어요

마치.. 정의되지 않은 변수.. 

그래서 결국 인터넷의 도움을 받았습니다 조건문 없이도 결과를 다르게 출력할 수 있는 방법!

**객체 리터럴 매핑 방식**이 있더라구요! 

객체에 키와 값을 선언하고 매개변수로 받은 값을 대조해 값을 출력하는 방식

**해당하는 값이 없을때 || (OR 연산자)를 통해서 "앞의 값이 없으면(false) 이면 뒤에 있는 값을 사용해!" 라고 기본 값을 설정할 수도 있다**는 것도 배웠어요

결론 : 객체 리터럴 매핑 방식을 통해 회원등급 객체 속성이 vip 일 때 대응되는 값인 메시지를 출력할 수 있었다. 
값을 대조해 보았을때 일치하는 값이 없으면 || 연산자 뒤쪽의 값이 출력된다!

### 1번 문제 테스트 코드 
```//1. welcomeMessage() 테스트 코드
console.log(welcomeMessage('홍길동', 'gold')) // '안녕하세요, 홍길동님! 즐거운 쇼핑 되세요.'

```

### 3번 문제 

이번 문제는 그래도 빠르게 풀었어요 앞에서 연습했던걸 써먹으니 금방 해결했습니다! 

문자열의 길이를 반환하는게 어떤게 있는지 궁금해서 인터넷에 검색해보고 length를 사용하면 길이를 반환할 수 있다는걸 알았습니다

그런데 문자열을 입력하지 않고 숫자형을 입력하면 반환되지 않아서 비밀번호를 String으로 변환해서 변수에 저장했어요

그리고나서 길이를 비교하고 결과를 보는데 빈 문자열일때도 true로 나오는걸 해결하고싶어서 

유효한 입력인지 확인하기 위해 비밀번호로 들어오는 문자열이 true 이면서 길이를 만족하는 경우를 비교식에 넣었더니 성공적으로 결과가 나왔습니다!

결론 : 문자형변수.length 를 사용하면 문자열의 길이를 반환할 수 있다는 사실을 배웠습니다!

### 3번 문제 테스트 코드
```
// 3. passwordValidation()
console.log(passwordValidation('1234567aabc')) // true
console.log(passwordValidation(12345)) //false
console.log(passwordValidation(123456789123456789123)) //false

```
